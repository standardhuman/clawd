### NTransformer ‚Äî Run Llama 70B on a Single RTX 3090
- **Source:** https://github.com/xaskasdf/ntransformer (Show HN, 215 points)
- **What it does:** C++/CUDA inference engine that streams model layers through GPU memory via PCIe, with optional NVMe-to-GPU direct I/O bypassing CPU entirely. Runs Llama 3.1 70B Q4_K_M at 0.5 tok/s on a single RTX 3090 (24GB VRAM) using 3-tier adaptive caching (VRAM + pinned RAM + NVMe). Also does 8B Q8_0 at 48.9 tok/s fully resident. Layer-skip via cosine similarity calibration eliminates 20/80 layers per token.
- **How we'd use it:** The 70B speeds aren't practical yet (0.5 tok/s), but the architecture ‚Äî 3-tier caching, layer skip, self-speculative decoding ‚Äî points to where consumer-hardware inference is heading. The 8B resident performance (49 tok/s) is already competitive with llama.cpp. Worth monitoring for when PCIe Gen5 NVMe doubles the bandwidth bottleneck.

### Claude Code Planning Workflow (Boris Tane Method)
- **Source:** https://boristane.com/blog/how-i-use-claude-code/
- **What it does:** Six-phase workflow: (1) Research ‚Äî deep-read codebase, write findings to research.md; (2) Plan ‚Äî detailed implementation plan with code snippets in plan.md; (3) Annotate ‚Äî human adds inline notes, Claude revises (repeat 1-6x); (4) Todo List ‚Äî extract actionable checklist; (5) Implement ‚Äî Claude executes against the plan; (6) Feedback. Key techniques: use strong language ("deeply", "intricacies") to prevent surface-level reading; reference implementations from open-source repos; markdown files over built-in plan mode.
- **How we'd use it:** This maps directly to OpenClaw's AGENTS.md workflow. The annotation cycle (human reviews plan, adds notes, agent revises) could be formalized in Howard's task delegation process. The "research.md ‚Üí plan.md ‚Üí todo ‚Üí implement" pipeline is worth adopting as a standard pattern for complex tasks.

## ⚓ SailorSkills Augmentation

### Clinejection: AI Supply Chain Attack — Security Audit Needed
- **Source:** https://grith.ai/blog/clinejection-when-your-ai-tool-installs-another (395 pts, 97 comments)
- **Summary:** An attacker injected a prompt into a GitHub issue title that tricked Cline's AI triage bot (running claude-code-action) into executing malicious code. The attack chain escalated through GitHub Actions cache poisoning to steal npm publishing credentials, resulting in cline@2.3.0 being published with a postinstall hook that installed OpenClaw on ~4,000 machines. The vulnerability was responsibly disclosed 5 weeks before exploitation but Cline failed to respond. Simon Willison covered this extensively, noting the pattern of "AI installing AI" as a new supply chain risk class.
- **Action:** Brian should audit any GitHub Actions workflows that use AI agents (claude-code-action, etc.) to ensure: (1) untrusted input (issue titles, PR descriptions) is never interpolated into agent prompts, (2) CI/CD secrets are isolated from triage workflows via separate cache keys, (3) any AI triage actions run with minimal permissions. This is also relevant content for a security-focused blog post about running AI agents safely.

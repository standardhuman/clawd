### Pi Agent ‚Äî The Minimal, Extensible Coding Harness
- **Source:** https://pi.dev (320 pts on HN)
- **What it does:** Pi is the underlying agent harness that powers OpenClaw, now available as a standalone tool. Provider-agnostic (15+ providers), extensible via TypeScript extensions, supports skills, prompt templates, tree-structured sessions, and four modes (interactive, print/JSON, RPC, SDK). Key design philosophy: "primitives, not features" ‚Äî sub-agents, plan mode, permission gates are all built as extensions, not baked in.
- **How we'd use it:** The HN comment from a developer who built a multi-tenant OpenClaw alternative using Pi's plugin system is revealing: "the most of [OpenClaw's] success comes from the underlying harness, pi agent." For Brian's development work, Pi's SDK mode could be used to build custom agent workflows without the full OpenClaw stack. The extension system (hooks at every stage from tool calls to compaction) is more flexible than Claude Code's SDK.

### Emdash ‚Äî Open-Source Agentic Development Environment
- **Source:** https://github.com/generalaction/emdash (143 pts, Show HN)
- **What it does:** Desktop app for running multiple coding agents in parallel, each isolated in its own git worktree, locally or over SSH. Supports 21 coding agent CLIs (Claude Code, Codex, Gemini, etc.). Key innovation: pre-allocated worktree reserve for ~500ms task startup. Integrates Linear/GitHub/Jira issues, diff review, CI/CD checks, and merge ‚Äî all from within the app.
- **How we'd use it:** If Brian does AI-assisted codebase migration consulting (from the Ladybird opportunity reported yesterday), Emdash's parallel agent orchestration + git worktree isolation is the exact tooling needed. Run Claude Code on module A, Codex on module B, compare results. The 21-provider support means you can match models to tasks.
